@Bean annotation in Spring Framework

What is the use of @bean annotation

@Bean annotation are used with the methods to produce beans. Spring Container will produce beans from a method when the method is annotated with @Bean annotation.

Spring @Bean methods can be created in @Configuration and @Component classes.

Methods can be annotated with @Bean by default bean name can be the method name annotated with @Bean. We can give our own name to the bean method...

@Bean methods at/within @Configuration class

In Spring application configurations can set in the class which annotated with @Configuration

initMethod and destroyMethod attributes of @Bean

initMethod attribute of @Bean specifies the intitialization method of a bean class/object. destroyMethod attribute of @Bean is for specifying the destroy method of a bean.

Initialization method can be executed at the time object retrieval and destroy method can be executed at the time of object deletion.

@Score of a @Bean

BY default all the beans initiated are singleton scope in Spring Framework

There are different types of bean scopes

@Scope("singleton") by default every bean is Singleton in spring
@Scope("prototype") this is a non singleton bean
@Scope("session")
@Scope("application")
@Scope("websocket")

Inner-Bean reference with @Bean annotation

Inner-Bean reference is possible only with @Configuration based @Bean methods, that means we must create/initiate bean with in @Configuration class.

Inner-bean reference is not support if bean initiated with in @Component class.
