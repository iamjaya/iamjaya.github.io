BeanFacotry vs ApplicationContext in Spring Framework

Every Spring based application required configuration details. These configuration details are using by Spring container to make application development and production process easy.

BeanFacotry introduction

BeanFacotry is a subset of ApplicationContext. BeanFacotry provides less functionalities compared with ApplicationContext.

ApplicationContext : for providing configuration details to the Spring Container

ApplicationContext is providing more functionalities compared to BeanFacotry. ApplicationContext is useful for providing application configuration details to the Spring Container.

If you want to access ApplicationContext in your bean (other bean) then we need to implement ApplicationContextAware interface.

To access non-default constructor of a bean in another bean is possible by using ApplicationContextAware only
